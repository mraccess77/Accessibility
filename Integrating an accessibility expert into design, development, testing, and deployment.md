Assumptions 
* Start from accessible design system
* Start from accessible framework and libraries
* Deveopers have access to automated accessibiliy testing tools

Accessibility Team Members
* Peform design review of wireframes, interactions, and core workflows
* Share feedback from design, adjust, and review again as needed
* Test a representative samples of pages as they are ready
* Provide feedback as it's available
* Validate fixes as groups of fixes are complete
* Continue through all types of pages have been tested and validated
* Automatically test all pages available
* Run use cases as soon as core tasks are available
* Validate fixes to use cases as the use case is fixed
* Regression test whole site manual and automatic and all use cases
* Validate final fixes
* Deploy monitoring solution with crawls and analytics for real time monitoring
* Repeat process for new content
